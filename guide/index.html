Temple is a lightweight view library for building complex web interfaces quickly and consistently.

Temple is most easily described as a view framework, similar to those commonly found in the MVC architecture. In reality, it is just a set of core JavaScript classes that either thinly wrap DOM nodes or provide interface control flow. Temple attempts to compliment the DOM, going so far as to match many of the DOM methods and properties. Unlike other view libraries, Temple was not built to replace the DOM but to provide structure for an already powerful API. The result is a flexible framework that feels natural and is fast as hell.

Temple is completely data agnostic, making it very compatible with existing systems. Reactivity in Temple is powered by a dependency-driven computation library that allows the interface to respond to changes consistently over time. This is the same library found in Meteor and for a more in-depth guide, check out the Deps Manual.

This guide will walk through the basics of Temple as well as some obscure API usage. For a reference on the available classes, methods and properties found in Temple, see the API Documentation.
