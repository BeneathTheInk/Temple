Events

Included in the Temple core is the event API taken directly Backbone. Below is a quick reference for the event API. Please see the Backbone documentation on events for a more in-depth guide.

view.on( event, callback [, context ])
view.once( event, callback [, context ])
view.off([ event ][, callback ][, context ])
view.trigger( event [, args... ])
view.listenTo( other, event, callback )
view.listenToOnce( other, event, callback )
view.stopListening( other, event, callback )

Instances of Temple will emit several events:

  • "parent" (previous, current) - when the parent binding of the view changes.
  • "child:add" (child) - when a child is added to the view.
  • "child:move" (child) - when an existing child binding changes positions.
  • "child:remove" (child) - when an existing child binding is removed from the view.
  • "update" - when updateNodes() is called on the view.
  • "paint" (parentNode, beforeNode) - when paint() is called, after it has been added to the DOM.
  • "detach" - when detach() is called, after it has been removed from the DOM.

Mount Life Cycle

Every Temple binding can initiate and run it's own reactive computation. While mounted, the binding will trigger events for each step in the cycle. The following is a list of those events and when in the cycle each is fired. Please see the mount() method for an in-depth look into how this cycle works.

  • "mount:before" (args) - Triggered immediately when the mount() method is called. Only passed args, an array of arguments passed to mount().
  • "render:before" (args, comp) - Triggered when the autorun computation begins. This means it is run once immediately with mount() and then zero or more time later. comp is the current autorun computation.
  • "render" (args, comp, bindings) - Triggered after render() is called on the binding, but before the bindings are added as children. bindings is the array of bindings returned from render()
  • "render:after" (args, comp, bindings) - Triggered after bindings from .render() are added as children.
  • "mount" (args, comp) - Triggered after the autorun's computation's first run.
  • "mount:after" (args, comp) - Triggered directly after the "mount" event.
  • "invalidate:before" (args, comp, bindings) - Triggered when the autorun computation is invalidated, but before the children bindings are removed.
  • "invalidate" (args, comp) - Triggered when the autorun computation is invalidated and the children bindings are removed.
  • "invalidate:after" (args, comp) - Triggered directly after the "invalidate" event.
  • "stop" (args) - Triggered when the autorun computation is stopped. This is always triggered after the "invalidate" event.
